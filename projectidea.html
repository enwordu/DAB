What is NFC?

An NFC Chip is a small device that allows transfer of data wirelessly. Its small nature and speedy communication would benefit the ‘Tap and Go’ feature of public transports we see today. The chip or tag stores data which communicates to an appropriate NFC Device, a reader or writer. Each tag is able to store data, however the size depends on the type of tag and individual capacity. NFC Chips are regarded as passive devices which means that they do not carry a power source within them. Instead they rely on the reader to provide power on the spot, the supplied power which comes from contact will provide enough time for the Chip to read and write or rewrite information depending on the data sent by the device. NFC Tags range in 4 types each with its own limitation, specification, and price.

Tag 1 is based on the ISO14443A standard, and is mainly used to store very small types of data such as website URLs. Its capacity is 454 bytes, without any possibility of modification. It’s communication speed is on average 106 kbit/s and is regarded as the cheapest tag amongst the. Due to its age, tag 1 is regarded to be the slowest chip amongst the 4. Its main function is to only Read or Write.

Tag 2 roughly follows tag 1’s standard, ISO14443A. It’s main difference lies on the memory. The capacity of a Tag 2 chip varies from 48 bytes to 2 kbytes. The price also varies depending on the capacity within the chip the larger capacity, the more expensive it would be. The speed of a Tag 2 chip mirrors a Tag 1 chip of, 106 kbit/s. Even if the price ranges depending on capacity, the tag 2 chip is still also regarded as one of the cheapest compared to the 4, and the better option due to its flexibility compared to Tag 1. Compared to the Tag 1, the Tag 2 is able to Read or Write, and Read only.

Tag 3 is where a difference is clearly shown not only by specifications but by function as well. This tag is based on the FeliCa system and has an available memory of 2 kbytes. It’s communication speed between devices is 212 kbit/s which doubles when compared to Tag’s 1 and 2. Similar to tag 1 and 2 its function can be used to either Read or Write, or Read-only. Due to its high capacity the price of a Tag 3 is regarded as the highest amongst the 4 types.

Tag 4 brings the best of both worlds between tags 1, 2, and 3. It is compatible with ISO14443A and B standards, and has a memory of 32 kbytes. It’s communication speed varies depending on model from 106 kbit/s to 424 kbit/s. It’s functionality mirrors tags 2 and 3 with, Read or Write, or Read-only. The main selling point of a Tag 4 is in the price and security. The cost of a tag 4 is considerably cheaper than a tag 3, and the encryption provided is a lot safer.

NFC encoder system.

NFC chips are comprised of 3 main components;

  • Microchip: To process data requests and store small amount of data

Why java?

Java is the official language used to develop android applications and for good reasons too. These reasons include; it being easy to learn, platform-independent and java is also object-oriented (IBM, 2018). Furthermore, the Android SDK (Software Development Kit) is written in Java and includes many useful resources for app development (Android Developers, 2018) such as; a debugger, standard Java libraries including data structure, maths, graphics, and networking libraries. And more importantly Android libraries which is particularly useful for app development. As well as sample codes and tutorials. Aside from Java being easy to learn due to the abundant availability of resources it is also one of the most human readable languages (Conder and Darcey, 2010). In that beginners are able to pick up Java easier and more quickly compared to other programming languages. The human readable Java source files are converted to bytecode via Java compilers, the codes are converted in order to allow the Java Virtual Machine to execute the compiled code. This is different to other programming languages in that they are not as portable. For example, the Dalvik Virtual Machine which is used to run Android applications allows the app to run regardless of what device, it could be any Android running device.

Object oriented programming or OOP is a form of programming language that depends on defining data structures and its functions as ‘objects’ (Beal, 2018). These objects that contain both data and function can interact with other objects if programmed to do so. There are many advantages of Java being a OOP language, a couple include; a modular structure which makes it easy to hide implementation details when one wishes to define abstract datatypes. Another advantage for having a modular structure is that it is easier to maintain for instances where there are issues, in that part of the system can be updated without having to make large-scale changes (Puri, 2017). Several other benefits include; faster development, can be easily modified by the programmer and better productivity (Puri, 2017). All in all, there are many reasons as to why using Java to program Android applications would be a good if not the ideal language to use. It is also the most sought out programming language by employers.

Why React Javascript

React JavaScript is a programming language invented by Facebook back at 2013. Its popularity grew over the years through its user-friendly and app-friendly designs. Currently it is well known for delivering great performance, with internal flexibility. This means that applications developed using React JavaScript and its interface are ‘Live’ and ‘Hot Reloading’. ‘Live’ and ‘Hot Reloading’ gives the freedom for developers to make changes on the spot while the application runs beside it. This eliminates the need to turn any device on and off to make changes.

One of the main downfalls of React JavaScript is its needy requirements. Compared to other development teams which utilize languages such as Objective-C or Swift for the IOS platform, or Java and Kotlin for Android, React JavaScript is known to be the most time consuming amongst the rest. Ultimately this also relates to the amount of work the team will have to invest, and the cost. Due to its friendly cross-platform abilities we have chosen React JavaScript as our end-stage language to integrate the public transport application into other platforms such as IOS. However by adopting React JavaScript, we believe extra blockades will appear due to its time and cost. It’s simple user interface also allows the developer to view the applications internal script, which makes it easier to locate and resolve errors within the system.

Initial Stage

This stage revolves around the creation of our prototype chip in hopes to gain investors to fund future stages, and hopefully a small implementation into certain rural areas within Australia for testing purposes. For this stage we will be programming an Android application which will be used on mobile phones. Recent developments in the hand held industry has led to an implementation of NFC chips inside phones which we can use in conjunction to our application. This would mean that people are able to touch on and off with their phones without the need of taking out a card. Ultimately however we would opt to export the data from the phone into an NFC chip which would placed inside a card. The type of NFC tag used for this stage would be Tag 3. The reason for using Tag 3 despite the cost and reduced encryption system is the flexibility it provides across multiple readers. Ultimately however on a large scale we would want to use a Tag 4 instead.

The programming language we would use to develop our application and code our chips is Java. The reason behind using Java is that it is known to be very friendly with Android applications. The limitation to this is that we would be heavily limited to the Android platform alone compared to IOS systems.
